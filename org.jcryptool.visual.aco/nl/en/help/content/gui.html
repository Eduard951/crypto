The user interface

Here the user interface is explained more precisely.

The screenshot below shows the view of the plug-in when initially loaded. In the upper and lower components brief explanations of the parameters and the view can be found. The centered component is the controller of the plug-in.

Configuration

First of all, a ciphertext has to be chosen in the Configuration panel. There are two possibilities to do so:

  • Generate ciphertext - A wizard will be started. The wizard helps you to generate a ciphertext from a plain text by encrypting it with a transposition cipher.
    Therefore, a ciphertext can be loaded from a file, from an opened editor window in JCrypTool or the text can be inserted directly. Next the length of the key of encryption can be chosen. This number indicates in how many columns in the lower table the text will be divided. Via drag'n'drop the table columns can easily be swapped and the key of encryption can be changed.
    By pressing on Finish the encoded text will be loaded to the Ant Colony Optimization plug-in.
  • Insert ciphertext directly - Alternatively, a pre-encoded text can be inserted in this text field.
    Information: Only upper case characters from A-Z are allowed.

Two important settings are:

  • Estimated key length - Sets the length of key which will be used for the decryption. The number coincides with the number of knots in graph. If you do not know this length, statistic test, e.g. the Friedman test, in the JCrypTool Menu: Analysis > Friedman test might help you.
  • Language of text - You can choose between German and English. The output of the algorithm will be compared and rated based on the frequency of common word in the chosen language. Therefore a dictionary will be used. Further on it is possible to provide your own dictionary by selecting "Load list of words from file".

After completing choosing those setting, the analysis can be started.

Information: The chosen text will automatically be completed such that the length of text is divisible by the key length. If not, it would give a hint of the position of a knot in the encryption, as some knots would hold less letters. Therefore, it is necessary that the length of the cipher is greater than the key length to start the analysis.

Analysis

The group Analysis handles the iterations of the algorithm. As in the Ant Colony Optimization Algorithm ants are sent through the graph, it is possible to control the ants step by step or directly send multiple ants. The Level of detail allows you to choose whether to send one ant from knot to knot or directly send multiple ants on their ways.

  • Knot-by-Knot - One single ant can be send and observed on its way.
    Information: This setting is active only if the length of key is less than 6 and Visualization as graph is chosen.
    By choosing so, the following options appear:
    • Animation - Animations in the graph can be enabled.
    • To next knot - By doing a Monte-Carlo selection based on the probability which is shown next to the graph, the ant selects its next knot to visit. How the probabilities are calculated can be found here.
    • Until last knot - It is the same as pressing To next knot until the ant is at the last knot.
    • Place new ant - If an ant has finished its path, a new ant can be placed. The ant will be placed randomly on one of the knots. The two buttons To next knot and Until last knot are active again, such that the new ant can be controlled.
  • Multiple iterations - Multiple ants can be sent at once. By selecting this option, the pheromone matrix will be shown automatically and following controllers appear.
    • Send new ants - Number of ants many ants will be sent through the graph at once. On the right side in the pheromone matrix you can observe what happens.
    • Number of ants - Determines the number of ants which will be sent at once.

The effects of the Algorithm settings can be observed in the probabilities which are shown next to the knots in the graph or in the pheromone matrix. To see how those values are calculated have a look here. The three parameters Alpha, Beta and Evaporation adjust the parameters explained there. The rule of priority \(\nu_{ij}\) is determined by the upcoming bigrams, which appear by concatenating the knots and its letters.

  • Alpha - On each iteration an ant drops a pheromon on the passed edges. Following ant are influenced by the pheromon and will more likely chose edges with a higher value of pheromon. Alpha determines the influence of the pheromon have over the ants.
  • Beta -This parameter adjusts the weight of the probabilities of the bigrams of letters which arise along the edges. By choosing a big parameter, the ant will choose a path where the bigrams of letters in the chosen language are more likely.
  • Evaporation - A small value means, that more following ants will notice the pheromon dropped on the edges by the current ant.

Visualization and result

If the key length is smaller than 6 and the option Knot-by-knot is activated, in Visualization you can choose between showing the graph or the pheromone matrix.

  • Visualization as graph - The appropriate graph with the current ant on its position and its path will be shown. In the graph, each knot contains letters (maximal 5 letters are displayed) of the ciphertext. If the ant has not yet reached its final position in the graph, next to each knot transposition probabilities are shown. Those probabilities represent the bigrams of letters which would arise by concatenating the knots. A higher probability therefore means, that the ant would more likely choose this knot.
  • Show pheromone matrix - The pheromone matrix will be shown. The matrix represents weight of edges in the graph. The matrix is symmetric and quadratic, the number of columns and rows equals the length of the key. Each entry \(a_{ij}\) correspond to a weight of the edge connecting Vertex \(i\) and \(j\). As in our graph there should not be an edge from each knot to itself, the diagonal values are not available. The value itself depends on how many ant have previously chosen this edges on their trail and it depends on the probability on the bigrams as well.

The Result component summarizes the overall results of the algorithm.

  • Number of ants sent so far - Displays the number of ants which have already passed the graph.
  • Current decryption - Shows the result of the current ant. The plain text and the key of the decryption of the ant is shown. The plain text can be found by concatenating the letters in the knot along the path (the key of decryption) the ant chose.
  • Best known decryption - Depending on how many known words are in the plaintext (known words are word which were found in the dictionary of the Language of text) the result of each ant is rated. The result which has the best grading is displayed here.
