Tutorial for the Dragon cipher

This tutorial will guide you through the process of an encryption and a decryption with a modern stream cipher, the Dragon cipher.
Note: As with all algorithms here, you can only perform the following steps when you have an active editor window.

Selecting Dragon from the menu or from the Crypto Explorer

Selecting the menu item Dragon in the menu path Algorithms â†’ Symmetric will bring up the wizard for the Dragon cipher. This will be applied to the active editor content (unsaved001.txt in the screenshot above).

Setting the parameters in the Dragon dialog for encryption

Select the size of key and IV: 128 bit or 256 bit. Then enter the according values for key and IV. The key and the IV have to be input in the selected format (hex or binary). In our example, we use 128 bit as our key/IV size and hexadecimal as the input format for both inputs. Our key has all 128 bit set to "1", and our IV has the value of 1 (i.e. 127 leading "0" bit and one "1" bit). Clicking on the Finish button will start the encryption process.
Note: Modern ciphers work on bits / bytes, but not on characters. So their output is always displayed in a hex editor rather than a text editor.

Performing the decryption

The resulting ciphertext (in output001.bin) can be decrypted, by selecting exactly the same parameters as for encryption (as it is a symmetric stream cipher): identical key / IV size and according values in the same wizard that was used for encryption.

Clicking on the Finish button will now start the decryption of the ciphertext and the correct plaintext is displayed in a new hex editor (output002.bin in the screenshot below).

Outputting the keystream

If you use – in the grouping "Output options"– the second radio button Display output file and keystream in separate editors then 2 new hex editor tabs are created: One contains the generated keystream from that encryption / decryption process, the other one contains either the encrypted plaintext or the decrypted ciphertext. A comparison of the keystream from the encryption process to that of the decryption process shows that – given identical input parameters – an identical keystream is produced.

The keystream is displayed in a new editor (output004.bin above), with the plaintext being displayed as usual (output003.bin).

Selecting the radio button Display keystream only, the specified number of bytes of the generated keystream is shown in another hex editor tab. The input area for the number of keystream bytes to be generated is only enabled if the radio button Display keystream only is selected.
